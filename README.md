# ◉ 세상을 보는 눈 (sebonoon)

뉴스를 많이 보여주는 사이트가 아닙니다. 신뢰 가능한 복수 출처에서 사건을 수집해
**① 무슨 일이 일어났고 ② 무엇이 확인된 사실이며 ③ 왜 중요한지 ④ 한국과 나에게 어떤 영향인지
⑤ 무엇이 불확실하고 ⑥ 무엇을 지켜봐야 하는지**를 한 페이지로 읽고,
예측을 기록·검증하며 '보는 눈'을 기르는 개인 대시보드입니다.

주소: https://sebonoon.koy.kr (GitHub Pages)

## 화면 구성 (위에서 아래로)

| 3 | 오늘 반드시 알아야 할 5가지 | 사건 카드 — 사실·의미·한국 영향·확실성 배지·출처 수·출처 펼쳐보기. 정보가 없는 블록은 숨김 |
| 4 | 어제 이후 달라진 내용 | 새로 등장한 사건 목록 |

| 11 | 대중 관심 신호 | Google Trends 한국 급상승 검색어 — **뉴스가 아닌 신호**로 분리 표시 |
| 12 | 개발자 커뮤니티 관심 신호 | Hacker News 인기 글 — **뉴스가 아닌 신호**로 분리 표시 |

보조 기능: ⭐ 관심 키워드(전 섹션 매칭, localStorage), 🔎 전 섹션 실시간 검색,
🌐 영어 제목 자동 번역(MyMemory 무료 API, 캐시), 라이트/다크 테마.
모두 **무료 공개 API/RSS**만 사용하며 API 키가 필요 없습니다.

## 출처 정책 — 공신력과 중립성

허용 출처는 **`scripts/lib/sources.mjs`의 등급 레지스트리가 단일 기준**이며, 이 문서·화면 문구·코드가 같은 내용을 말합니다.

| type | 의미 | 등록 매체 |
|---|---|---|
| `wire` | 통신사 (사실 보도 중심) | 연합뉴스(tier 1) · 뉴시스 · 뉴스1 · 연합인포맥스 · Reuters(tier 1) · AP(tier 1) |
| `public` | 공영방송 | KBS(tier 1) · YTN · BBC(tier 1) |
| `specialist` | 과학·기술 전문 | Science(AAAS) · Nature · MIT Technology Review · CoinDesk |
| `primary` | 당사자(정부·기관) 발표 채널 | KTV — **사실의 최종 기준이 아니라 공식 '주장'으로 취급** (`primary-claim` 배지) |
| `signal` | 관심 신호 (뉴스 아님) | Hacker News · Google Trends |
| `unknown` | 미검증 | 집계에서 제외 |

핵심 규칙:
- Google News 피드는 화이트리스트 통과분만 사용합니다. **통과 기사가 부족해도 기준을 낮춰
  임의 매체를 포함하지 않습니다** — 1건이면 1건, 0건이면 빈 목록이며, 부족 상태는
  `sectionMeta`에 기록되어 화면에 '신뢰 출처 보도 부족'으로 표시됩니다.
- **가독성을 위해 국제 뉴스도 한국어 보도를 우선**합니다(`koFirst`) — 한국어 소스로 먼저 채우고,
  부족분만 원문 국제 소스로 보충하며 영어 제목은 자동 번역됩니다.
- 스포츠·연예·단순 생활 기사는 핵심 브리핑에서 제외합니다 (`scripts/lib/classify.mjs`, 규칙 분리 — 향후 별도 카테고리 확장 가능).
- 정파적 성향이 뚜렷한 매체는 집계하지 않습니다.
- 시장 데이터: ECB 고시환율(Frankfurter) · CoinGecko · Yahoo Finance — 1차 데이터.

## 동작 방식

1. `.github/workflows/daily-collect.yml` — GitHub Actions가 **하루 4회(KST 06:17 · 12:17 · 18:17 · 23:17)**
   `scripts/collect.mjs` 실행 (예약 실행은 부하에 따라 수 분 지연될 수 있는 best-effort, 수동 실행 가능)
2. 수집기는 기사들을 **사건 단위로 클러스터링**하고(제목 토큰·고유명사·발행 시각), 사건마다
   **설명 가능한 중요도·신뢰도 점수**(계산 이유 포함)와 **규칙 기반 해석 브리핑**
   (확인된 사실 = 출처 제목 그대로 / 의미·한국 영향·주시 포인트 = 키워드 규칙, 추정은 '가능성·예상·확인 필요' 표기)을 생성합니다.
   출처에서 확인되지 않는 내용은 생성하지 않으며, 규칙으로 만들 수 없는 해석은 비워 둡니다.
3. **품질 게이트**(`scripts/lib/quality.mjs`)를 통과한 스냅샷만 `data/latest.json` + `data/last-good.json`으로
   승격됩니다. 불합격이면 기존 latest.json(마지막 정상 데이터)을 유지하고 `data/status.json`에 사유만 기록 —
   화면 상단 상태줄이 정상/일부 실패/수집 지연을 구분해 보여줍니다.
4. `data/history.json` — 지표를 **일중 OHLC**(open/high/low/close/samples)로 누적 (최근 120일, 하루 4회 수집이 고가·저가로 병합됨)
5. `data/news-archive.json` — 사건 ID·제목·출처·URL·발행 시각·신뢰 등급 보존 (최근 60일, 같은 날짜는 덮어쓰지 않고 누적)
6. `index.html` + `assets/`(ES 모듈, 번들러 없음)가 데이터를 렌더링 — 환율·코인 시세는 브라우저에서도 조회

### 과부하 방지 캐시 (30분)

브라우저는 데이터 파일과 외부 시세 API(CoinGecko·Frankfurter) 응답을 **localStorage에 30분간 캐시**합니다.
새로고침을 반복해도 30분 안에는 재요청하지 않고, 네트워크 실패 시 만료된 캐시라도 사용합니다(stale fallback).
뉴스는 하루 4회 수집이므로 시세만 30분 주기로 갱신됩니다.

## 시작하기

```bash
npm test                   # 테스트 (node:test — 의존성 설치 불필요, Node 20+)
node scripts/collect.mjs   # 데이터 수집
npx serve .                # 정적 서버로 열기 → http://localhost:3000 (ES 모듈이라 file:// 로는 열리지 않음)
```

fixture로 네트워크 없이 수집기를 실행하려면:
`SEBONOON_FIXTURES=<fixture 디렉터리> SEBONOON_DATA_DIR=<임시 디렉터리> node scripts/collect.mjs`
(fixture 생성은 `test/helpers/fixtures.mjs` 참고 — 라이브 `data/`를 훼손하지 않고 테스트할 수 있습니다)

> 예약 워크플로우(cron)는 **기본 브랜치(main)** 에서만 동작합니다.

## 예측 판정 — 전부 무료, 외부 API 없음

| 유형 | 판정 방식 |
|---|---|
| 📊 지표 예측 | **자동.** 일중 고가/저가 기준 — `이상` 예측은 기간 내 `high ≥ 목표`, `이하` 예측은 `low ≤ 목표`면 적중. 기한이 지나도 **수집 데이터가 부족하면 자동 실패가 아니라 '데이터 부족 — 직접 판정'** 을 요청합니다. 자동 판정은 '정정'으로 뒤집기 가능 |
| ✏️ 자유 예측 | **직접 판정 + 무료 판정 도우미.** 예측 키워드를 기간 중 사건 아카이브와 대조해 관련 헤드라인(출처 링크 포함)을 보여줍니다 |

구버전 history.json(지표당 숫자 1개)은 자동으로 `o=h=l=c`로 해석되므로 마이그레이션 작업이 필요 없습니다.

### 선택: 로컬 LLM으로 자유 예측 자동 판정 — 무료

맥미니 등 내 기기에서 로컬 LLM 서버를 돌리고 있다면 자유 예측도 자동 판정할 수 있습니다.
브라우저 → 내 기기 직접 통신이라 **외부 서버 전송·비용이 전혀 없고, 로컬 서버를 외부에 공개하지 않습니다.**

판정 안전 규칙:
- 증거는 **예측과 관련된 기간 중 헤드라인만** 전달 (전체 아카이브 임의 샘플링 금지, 모델의 일반 지식 사용 금지 지시)
- 직접 근거가 있을 때만 적중, 명확한 반대 결과가 확인될 때만 빗나감 — **관련 보도가 없다는 이유만으로 빗나감 처리하지 않음**
- 응답 JSON 스키마 검증 — 어긋나면 폐기하고 수동 판정 유지
- 요청 타임아웃(AbortController) 적용, 실패 시 조용히 건너뜀

**두 API 방식을 자동 감지**하므로 서버 종류는 무엇이든 됩니다:

| 방식 | 해당 서버 | 감지 엔드포인트 |
|---|---|---|
| OpenAI 호환 | **oMLX**, mlx_lm.server, LM Studio, llama.cpp server 등 | `/v1/models`, `/v1/chat/completions` |
| Ollama 네이티브 | Ollama | `/api/tags`, `/api/chat` |

설정: 예측 노트 → **🖥️ 로컬 LLM 자동 판정** → 주소 입력 → **연결 테스트** → 저장.
CORS 허용 필요(Ollama: `launchctl setenv OLLAMA_ORIGINS "*"` 후 재시작, LM Studio: 설정에서 CORS 켜기).
HTTPS 페이지에서 `http://localhost` 호출은 Chrome/Edge에서 동작합니다.
한국어 판정이므로 한국어가 되는 모델 추천: EXAONE 3.5, Qwen 2.5(7B+) 등.

## 도메인 이전 (sebanoon → sebonoon)

localStorage는 도메인 간 자동 이동되지 않습니다. 이전 절차:
1. **기존 주소**(sebanoon.koy.kr)에서 예측 노트 → **내보내기 (JSON)** — 스키마 버전이 포함된 파일이 저장됩니다
2. DNS·GitHub Pages 설정 변경 (저장소 관리자 작업 — 완료 보고의 체크리스트 참고)
3. **새 주소**(sebonoon.koy.kr)에서 예측 노트 → **가져오기** — 중복 예측은 갱신되고, 잘못된 파일은 명확한 오류로 거부됩니다

## 테스트

```bash
npm test   # node:test — 출처 필터/RSS 파싱/분류/클러스터링/점수/품질 게이트/OHLC 판정/아카이브 검색/가져오기 등 28개
```

수집기 통합 테스트는 fixture(`test/helpers/fixtures.mjs`)로 전체 성공·부분 실패·전체 실패·허용 출처 0건 시나리오를 재현합니다.

## 보는 눈 훈련 루틴

1. 매일 아침 이 페이지를 5분간 훑는다 — TOP5부터
2. '오늘의 생각거리'에 스스로 답한다
3. 일주일에 2–3개, **기한이 있는 예측**을 확신도(%)와 함께 기록한다
4. 판정일이 되면 반드시 적중/빗나감을 판정한다
5. **적중률과 평균 확신도의 차이**(과신 여부)와 브라이어 점수를 보고 다음 예측을 보정한다

> 예측은 브라우저 localStorage에 저장됩니다. 기기를 옮길 땐 '내보내기/가져오기'를 사용하세요.
